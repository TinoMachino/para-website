PARA DEMO RUNBOOK
=================

Goal
----
Raise the full local demo with the current workspace split:

- website = docs / marketing site
- PARA = app client
- PARA/bskyweb = browser-facing web demo server
- watx = backend stack (PLC, PDS, AppView)


Before You Start
----------------
1. Open macOS Terminal.
2. Create or choose window profiles so you can visually separate each service.
3. Use these colors:
   - Terminal 1: Blue Ocean
   - Terminal 2: Blue Ocean
   - Terminal 3: Blue Ocean
4. If you want stronger separation, keep Blue Ocean for the client-facing app window and add a subtitle in the tab title:
   - WEBSITE
   - PARA EXPO
   - PARA BSKYWEB
   - WATX BACKEND


Terminal 1: WEBSITE
-------------------
Profile color:
- Blue Ocean

Commands:

cd /Users/mlv/Desktop/MASTER/website
pnpm install
pnpm dev

What this does:
- Starts the SvelteKit docs / marketing site.

Expected result:
- Website available on the local Vite dev port printed by the command.


Terminal 2: PARA EXPO WEB CLIENT
--------------------------------
Profile color:
- Blue Ocean

Commands:

cd /Users/mlv/Desktop/MASTER/PARA
yarn
yarn web

What this does:
- Starts the PARA Expo web client.

Expected result:
- Web app available on the local Expo web port printed by the command.


Terminal 3: PARA BSKYWEB FRONTEND
---------------------------------
Profile color:
- Blue Ocean

Commands:

cd /Users/mlv/Desktop/MASTER/PARA/bskyweb
go run ./cmd/bskyweb serve --appview-host https://appview.paramx.social.ngrok.pro --http-address :8100

What this does:
- Starts the browser-facing web demo server.
- This is the frontend entry you can open in Safari or send through a tunnel.

Expected result:
- Local browser entry at http://localhost:8100

Important:
- Do not send the AppView URL itself to demo users.
- AppView is backend infrastructure, not the browser-facing client.
- If you need a public demo link, tunnel or host port 8100 and send that frontend URL.


Terminal 4: WATX BACKEND
------------------------
Profile color:
- Blue Ocean

Commands:

cd /Users/mlv/Desktop/MASTER/watx
make nvm-setup
make deps
make build
make run-dev-env

What this does:
- Starts the backend development stack.

Expected local services:
- PLC: http://localhost:2582
- PDS: http://localhost:2583
- AppView: http://localhost:2584


Startup Order
-------------
Use this order:

1. Start Terminal 4 first so backend services are ready.
2. Start Terminal 3 second so the browser-facing web demo can point at AppView.
3. Start Terminal 2 third if you also want the Expo web client running.
4. Start Terminal 1 last for the docs / demo website.


If You Want The Shared Persistent Demo Instead
---------------------------------------------
Use this in Terminal 4 instead of make run-dev-env:

cd /Users/mlv/Desktop/MASTER/watx
cp -n .env.shared-demo.example .env.shared-demo
make run-dev-env-persistent

Shared public identities used by that runbook:
- https://pds.paramx.social
- https://appview.paramx.social

If you use the ngrok-based browser demo flow, the frontend still needs its own public URL.
Do not confuse these:

- AppView backend URL = data/API service
- bskyweb URL on port 8100 = actual browser demo


Quick Smoke Check
-----------------
1. Confirm Terminal 4 shows the backend services came up cleanly.
2. Open http://localhost:8100 from Terminal 3.
3. Confirm the browser demo loads without backend connection errors.
4. Open the PARA Expo web client from Terminal 2 if you also want that surface available.
5. Open the website from Terminal 1.
6. Verify the docs and any VC-facing link point to the frontend URL, not AppView.


If Something Breaks
-------------------
- If website fails: rerun pnpm install in /Users/mlv/Desktop/MASTER/website
- If PARA fails: rerun yarn in /Users/mlv/Desktop/MASTER/PARA
- If bskyweb fails: make sure Terminal 4 is healthy first, then rerun the go run command in /Users/mlv/Desktop/MASTER/PARA/bskyweb
- If watx fails: check Docker first, then rerun make deps and make build


Minimal Version
---------------
If you only care about the actual app demo and not the docs site:

1. Start Terminal 4
2. Start Terminal 3

You can skip Terminal 1 and Terminal 2.
