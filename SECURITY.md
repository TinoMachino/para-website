# Security Policy

## Supported Surface

This repository is for the public PARA website and documentation stack.

Security reports are especially relevant for:

- exposed secrets or credentials
- deployment misconfiguration
- script behavior that could publish unintended artifacts
- cross-site scripting or content injection risks
- dependency vulnerabilities that materially affect the public site

## Reporting a Vulnerability

Do not open a public GitHub issue for a security problem.

Until a dedicated security mailbox is published, report vulnerabilities privately to the repository maintainers through GitHub security reporting or direct maintainer contact.

When reporting, include:

- a concise description of the issue
- affected files or routes
- reproduction steps
- impact assessment
- any suggested remediation

## Disclosure Expectations

- Give maintainers reasonable time to investigate and patch.
- Do not publish exploit details before a fix or mitigation is available.
- If the issue touches private infrastructure outside this repo, keep those details private unless maintainers explicitly approve disclosure.
