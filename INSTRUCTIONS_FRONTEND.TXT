PARA FRONTEND DEMO RUNBOOK
==========================

Goal
----
Raise the browser-facing PARA demo and generate the correct link to send to other people.

This file is only about the frontend side.


What Counts As The Frontend
---------------------------
For this setup:

- AppView URL = backend data/API service
- bskyweb on port 8100 = actual browser-facing frontend

Important:
- Do NOT send the AppView URL to a VC or external tester.
- The only valid demo link is the frontend URL that opens the web client.


Terminal 1: FRONTEND SERVER
---------------------------
Suggested macOS Terminal profile color:
- Blue Ocean

Commands:

cd /Users/mlv/Desktop/MASTER/PARA/bskyweb
go run ./cmd/bskyweb serve --appview-host https://appview.paramx.social.ngrok.pro --http-address :8100

What this does:
- Starts the browser-facing PARA web client.
- Points that frontend at the AppView backend.

Expected result:
- Local frontend URL: http://localhost:8100


Local Demo Link
---------------
If you are only testing on your own machine, open:

http://localhost:8100


Public Demo Link
----------------
If you want a link you can send to a VC, tester, or anyone outside your machine, you must expose port 8100 publicly.

Example using ngrok:

Terminal 2:

ngrok http 8100

What to copy:
- Copy the public HTTPS URL that ngrok gives for port 8100.
- That URL is the real demo link.

Do NOT copy:
- https://appview.paramx.social.ngrok.pro

Why:
- That is backend infrastructure.
- It is not the browser-facing client.


Correct Link Model
------------------
Backend:
- https://appview.paramx.social.ngrok.pro

Frontend local:
- http://localhost:8100

Frontend public:
- whatever public HTTPS URL points to port 8100


Smoke Check
-----------
1. Start the frontend server.
2. Open http://localhost:8100 locally.
3. Confirm the app shell renders.
4. If using ngrok, open the public HTTPS tunnel URL.
5. Confirm the tunnel opens the same frontend, not a backend error page.

If It Fails
-----------
- If localhost:8100 does not open, rerun the go run command in /Users/mlv/Desktop/MASTER/PARA/bskyweb
- If the frontend opens but data fails, the AppView backend is the next thing to check
- If the public link fails but localhost works, the tunnel is the problem, not the frontend
